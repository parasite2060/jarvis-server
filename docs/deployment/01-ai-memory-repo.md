# Setup: ai-memory Repository

The ai-memory repo is Jarvis's brain — a git-tracked collection of markdown files that store your personality, decisions, and patterns.

## 1. Create the GitHub Repository

```bash
gh repo create ai-memory --private --description "Jarvis AI memory vault"
git clone git@github.com:YOUR_USERNAME/ai-memory.git
cd ai-memory
```

## 2. Initialize the Vault Structure

Copy the template from `docs/deployment/templates-ai-memory/` in the jarvis-server repo:

```bash
cp -r /path/to/jarvis-server/docs/deployment/templates-ai-memory/* .
```

Or create these files and directories manually:

```
ai-memory/
├── SOUL.md              # Worldview, decision principles, opinions, boundaries
├── IDENTITY.md          # Role, persona, tech stack, working style, projects
├── MEMORY.md            # Patterns, decisions, facts, recent session extractions
├── config.yml           # Dream configuration
├── _guide.md            # Vault structure reference
├── dailys/              # Daily session logs (auto-generated by dreams)
├── decisions/           # Decision records with reasoning
│   └── _index.md
├── projects/            # Active project context
│   └── _index.md
├── patterns/            # Reinforced behavioral patterns
│   └── _index.md
└── templates/           # Reusable prompts and templates
    └── _index.md
```

## 3. Create a GitHub Personal Access Token (PAT)

The server needs a PAT to push dream branches and create PRs.

1. Go to **GitHub Settings** > **Developer settings** > **Personal access tokens** > **Fine-grained tokens**
2. Click **Generate new token**
3. Configure:
   - **Token name:** `jarvis-server`
   - **Expiration:** 90 days (or custom)
   - **Resource owner:** Your account
   - **Repository access:** Only select repositories > select `ai-memory`
   - **Permissions:**
     - **Contents:** Read and write (clone, push, commit)
     - **Pull requests:** Read and write (create PRs, enable auto-merge)
     - **Metadata:** Read-only (required by default)
4. Click **Generate token** and copy it immediately

Save this token — you'll need it as `JARVIS_GITHUB_PAT` in your `.env`.

## 4. Clone on the Server

On the machine running Docker:

```bash
# Clone using HTTPS with the PAT
git clone https://x-access-token:YOUR_PAT@github.com/YOUR_USERNAME/ai-memory.git /opt/ai-memory

# Configure git user for dream commits
cd /opt/ai-memory
git config user.name "Jarvis Dreamer"
git config user.email "jarvis@yourdomain.dev"
```

## 5. Commit and Push the Initial Structure

```bash
cd /opt/ai-memory
git add -A
git commit -m "feat: initialize ai-memory vault structure"
git push origin main
```

## Verification

```bash
# Repo exists and has the vault structure
ls /opt/ai-memory/SOUL.md /opt/ai-memory/IDENTITY.md /opt/ai-memory/MEMORY.md /opt/ai-memory/config.yml

# Git can push (test with a no-op)
cd /opt/ai-memory && git pull origin main
```
